
# Análise de Requisitos para um Sistema de Gerenciamento de Biblioteca

### Introdução
A análise abrange requisitos não funcionais e casos de uso, fundamentais para a construção de um sistema robusto e eficiente que atenda às necessidades de bibliotecários e usuários.

# Justificativa da Análise de Requisitos
A análise de requisitos é uma fase crítica no ciclo de vida de desenvolvimento de software, especialmente para um sistema complexo como o de gerenciamento de biblioteca. A escolha de dedicar tempo e recursos significativos a esta etapa se justifica por diverosos fatores:

##### 1.Alinhamento de Expectativas: 
Garante que todas as partes interessadas (bibliotecários, administradores, usuários finais) tenham uma compreensão comu do que o sistema fará e como ele funcionará. Isso minimiza mal-entendidos e a necessidade de retrabalho em fases posteriores.

##### 2.Redução de Custos e Riscos:
Identificar e documentar os requisitos de forma clara e abrangente no início do projeto ajuda a evitar a construção de funcionalidades essenciais. Erros detectados nas fases iniciais são significativamente mais baratos de corrigir do que aqueles encontrados durante a implementação ou após a entrega do sistema.

##### 3.Base para o Design e Desenvolvimento:
Os requisitos funcionais e não funcionais servem como a espinha dorsal para as fases de design arquitetural, design de interface do usuário, desenvolvimento e testes. Um conjunto de requisitos bem definidos facilita a criação de um sistema que seja não apenas funcional, mas também utilizável, seguro e com bom desempenho.

##### 4.Validação e Verificação:
Fornece os critérios pelos quais o sistema final será avaliado. Cada requisito documentado pode ser rastreado e testado para garantir que o sistema entregue o que foi prometido.

##### 5.Melhoria da Comunicação:
Atua como um documento central que facilita a comunicação entre a equipe de desenvolvimento e os stakeholders, garantindo que todos estejam na mesma página em relação aos objetivos do projeto.

# Conclusão
Esta análise de requisitos fornece uma base sólida para o desenvolvimento de um Sistema de Gerenciamento de Biblioteca. Ao detalhar os requisitos funcionais e não funcionais, juntamente com os casos de uso, o documento serve como um guia para a equipe de desenvolvimento, garantindo que o sistema final atenda às expectativas e necessidades da biblioteca e de seus usuários. A adesão a estes requisitos contribuirá para a criação de um SGB eficiente, seguro e fácil de usar.
